## Overview

v2.9.0 is a foundational release that restructures the entire codebase from a monolithic architecture into a well-tested modular system, fixes longstanding scroll synchronization bugs, and adds new features — all without changing user-facing behavior of existing functionality.

### Highlights

- **Complete codebase restructuring** across 8 phases: type safety, pure function extraction, module separation, hook extraction
- **Scroll sync reliability**: Fixed feedback loops, idle disconnect false positives, and scroll drift
- **Panel position persistence**: Drag position now survives page reloads (per-tab isolation)
- **Suggestion toast fix**: No more false sync suggestions during active synchronization
- **Domain exclusion management**: Temporary snooze (2h) and permanent domain exclusion with full keyboard-navigable management UI
- **863 tests** covering all core business logic (unit, integration, and scenario tests)
- **Locale-preserving URL sync**: Multilingual documents keep their language settings during URL sync

## 1. New Features

### 1.1 Panel Drag Position Persistence

**PR**: #337

The floating sync control panel now remembers its drag position across page reloads within the same tab.

**How it works**:

- When you drag the panel to a new position, it saves to `sessionStorage` (tab-scoped)
- On page reload or SPA navigation, the panel restores to its last position
- Each tab maintains its own independent position — dragging in Tab A does not affect Tab B
- Position is validated against the current viewport on restore (prevents off-screen placement)

**Technical details**:

- Storage: `sessionStorage` with key `__sync_tab_scroll_panel_pos`
- Lazy `useState` initializer eliminates the initial-position flicker
- No cross-tab broadcast — previous implementation accidentally synchronized panel position across all synced tabs via `browser.storage.local`

### 1.2 Locale-Preserving URL Sync

When URL navigation sync is enabled, the extension now preserves each tab's locale (language code) during navigation.

**Example**:

```
Tab A navigates to: https://docs.example.com/fr/guide/install
Tab B (currently at: https://docs.example.com/en-US/guide/getting-started)
Tab B syncs to:     https://docs.example.com/en-US/guide/install  ← locale preserved
```

**Supported locale formats**:

- ISO 639-1 base codes: `en`, `fr`, `ko`, `ja`, `zh`, etc. (82 languages)
- Regional variants: `en-US`, `zh-CN`, `pt-BR`, etc.
- Position-aware: locale is inserted at the same path position as the source URL

**Behavior rules**:
| Source has locale | Target has locale | Result |
|---|---|---|
| ✓ | ✓ | Replace source locale with target locale, keep path |
| ✓ | ✗ | Use source URL as-is (including source locale) |
| ✗ | ✓ | Use source path (target locale is not re-applied) |
| ✗ | ✗ | Standard URL sync |

Target URL's query parameters and hash fragments are always preserved.

### 1.3 Auto-Sync Group Management

The auto-sync feature has been significantly improved with proper state management:

- **URL normalization**: Tabs are grouped by normalized URL (strips query strings, fragments, default ports, trailing slashes)
- **Mutex-guarded operations**: Concurrent URL change events cannot corrupt group membership
- **Exclusion patterns**: Configurable URL patterns to exclude from auto-sync grouping
- **Local dev server detection**: Automatically excludes `localhost`, `127.0.0.1`, and IPv6 loopback addresses

### 1.4 Add-Tab Suggestion

When a new tab opens with a URL matching an already-active manual sync, a toast appears offering to add the tab to the existing sync session — no need to stop and restart.

### 1.5 Domain Exclusion Management

**PRs**: #344, #345

Users can now control which domains show sync suggestion toasts, at two levels:

**Temporary snooze** (PR #344): Dismissing a suggestion toast ("Not Now" or "✕") suppresses suggestions for that domain for 2 hours. Stored in-memory only — resets on service worker restart.

**Permanent exclusion** (PR #345): Two ways to permanently exclude a domain:

1. Click "Don't show again for this site" directly on the suggestion toast
2. Manage excluded domains via Actions menu → "Manage Excluded Domains"

The management dialog features:

- **Smart URL parsing**: Paste any URL and the domain is extracted automatically (`https://github.com/user/repo` → `github.com`)
- **Real-time validation**: Shows extracted domain preview as you type, warns if already excluded
- **Keyboard navigation**: Raycast-like arrow key navigation with `aria-activedescendant` pattern
- **Two-stage Enter confirmation**: First Enter marks for deletion (visual warning), second Enter confirms — prevents accidental removal
- **IME support**: CJK input composition does not interfere with keyboard shortcuts
- **Focus restoration**: Search input automatically refocuses when dialog closes

See `docs/guides/domain-exclusion.md` for full architecture and implementation details.

## 2. Bug Fixes

### 2.1 False Disconnect Status During Idle Periods

**Symptom**: After a few minutes without scrolling, the badge changed from blue (connected) to orange (disconnected) even though sync was working fine.

**Root cause**: `lastSuccessfulSync` was only updated on `scroll:sync` events. The background keep-alive pings (every 25s) were arriving successfully but not updating the timestamp. After 60s without a scroll event, the health check declared the connection dead.

**Fix**: Update `lastSuccessfulSync` when `scroll:ping` messages arrive (1 line change). Now the health check correctly distinguishes "no scrolling" from "connection lost."

### 2.2 Scroll Drift / Feedback Loop

**Symptom**: "Synchronization is lost from the very beginning — one page is delayed, the more you scroll the more delay."

**Root causes** (two interacting issues):

1. **Async storage reads on every scroll event**: `handleScrollCore()` called `await getManualScrollOffset()` (1-10ms variable latency) on every scroll event — twice per sync cycle (sender + receiver = 2-20ms variable delay)
2. **Grace period race condition**: The 100ms grace period was too tight for the actual pipeline latency of 85-135ms, causing the receiving tab to broadcast back to the sender (feedback loop)

**Fixes**:

- **In-memory offset cache**: Replaced async storage reads with synchronous in-memory cache. Cache is updated only when the offset actually changes (Alt+scroll end, URL change, sync stop)
- **Grace period widened**: 100ms → 200ms, providing safe margin over the worst-case 135ms pipeline delay

### 2.3 Suggestion Toast During Active Sync

**Symptom**: Sync suggestion toasts appeared even when tabs were already actively syncing.

**Root causes** (three interacting issues):

1. **Race condition in `main.ts`**: `initializeAutoSync()` ran before `restoreSyncState()` completed, so `syncState.isActive` was `false` when checking
2. **In-memory state loss**: `manualSyncOverriddenTabs` (a `Set<number>`) was lost on MV3 service worker restart, causing synced tabs to be re-added to auto-sync groups
3. **Ping-only validation**: `showSyncSuggestion()` relied solely on content script pings (unreliable due to Chrome tab throttling) instead of checking background-owned `syncState`

**Fixes**:

- `await restoreSyncState()` before `initializeAutoSync()`
- Rebuild `manualSyncOverriddenTabs` from `syncState.linkedTabs` on restore
- Add `syncState.isActive` + `hasSyncedTabMatchingUrl()` check before ping-based validation
- Deduplicate add-tab suggestions in `onUpdated` handler
- Clean up `pendingSuggestions` on sync start/stop

### 2.4 Cross-Tab Panel Position Contamination

**Symptom**: Dragging the panel in Tab A caused Tab B's panel to move to the same position after page navigation.

**Root cause**: Two sources of contamination:

1. Real-time broadcast via `panel:position` message relay through background
2. Global `browser.storage.local` key shared across all tabs

**Fix**: Switched to `sessionStorage` (browser-native tab isolation). Removed all cross-tab broadcast logic. Each tab now has its own independent storage scope.

## 3. Performance Improvements

### 3.1 Synchronous Offset Cache

**Before**: Every scroll event (20 per second) triggered 2 async `browser.storage.local.get()` calls (sender + receiver), each with 1-10ms variable latency.

**After**: In-memory `cachedManualOffset` variable with O(1) synchronous access. Cache is updated only when the offset value actually changes:

- Sync start → load from storage
- Alt+scroll end → update from new value
- URL change / sync stop → reset to `{ratio: 0, pixels: 0}`

**Impact**: Eliminated 2-20ms variable latency per scroll cycle, making sync timing deterministic.

### 3.2 Reduced Bundle Size

Removing unnecessary imports from `use-drag-position.ts` (webext-bridge, webextension-polyfill, logger) after switching to sessionStorage reduced the content script bundle: 490.83KB vs 491.45KB.

### 6.2 Key Pitfalls (Prevent Regression)

1. **No `await` in hot path**: Scroll event handlers must use synchronous in-memory cache, never async storage reads
2. **"No events" ≠ "disconnected"**: Health checks must use connection-level signals (pings), not business events (scroll)
3. **Grace period > pipeline latency**: `PROGRAMMATIC_SCROLL_GRACE_PERIOD` (200ms) must exceed worst-case pipeline delay (~135ms)
4. **Cache sync discipline**: `cachedManualOffset` must be updated at every save/clear call site
5. **Timing constant invariants**: `KEEP_ALIVE_INTERVAL_MS × 2 < CONNECTION_TIMEOUT_THRESHOLD`
6. **No global storage for tab-local data**: Use `sessionStorage` for per-tab state, `browser.storage.local` only for global settings
7. **Service worker restart recovery**: In-memory `Set`/`Map` state must be rebuilt from persistent storage
8. **Background state before pings**: Check `syncState.isActive` before sending content script pings

### 7.2 Message Protocol

All messages are type-safe via `webext-bridge` ProtocolMap. Key message types:

| Message                     | Direction              | Payload                             |
| --------------------------- | ---------------------- | ----------------------------------- |
| `scroll:start`              | Popup/BG → Content     | `{ tabId, linkedTabs, mode }`       |
| `scroll:stop`               | Popup/BG → Content     | `{}`                                |
| `scroll:sync`               | Content → BG → Content | `{ scrollRatio, maxScroll, tabId }` |
| `scroll:ping`               | BG → Content           | `{}` → `{ isActive, tabId }`        |
| `scroll:manual`             | Content → BG → Content | `{ enabled, tabId }`                |
| `url:sync`                  | Content → BG → Content | `{ url, sourceTabId }`              |
| `sync:status`               | BG → Content           | `{ isActive, linkedTabs }`          |
| `auto-sync:suggest`         | BG → Content           | `{ tabs, groupUrl }`                |
| `auto-sync:add-tab-suggest` | BG → Content           | `{ tabId, tabTitle }`               |
| `snooze-domain`             | Content → BG           | `{ domain }`                        |
| `exclude-domain-permanent`  | Content → BG           | `{ domain }`                        |
| `get-excluded-domains`      | Popup → BG             | `{}` → `string[]`                   |
| `add-excluded-domain`       | Popup → BG             | `{ domain }` → `{ success }`        |
